Phase 1 新增 Crate 设计

6 个新 crate 的职责、公开 API、依赖关系

1. crates/inference/ — 模型推理

职责 & API
职责: 封装 llama-cpp-2，加载 GGUF 模型，提供文本补全

公开类型:
struct InferenceEngine { ... }
struct CompletionRequest { prompt, max_tokens, temperature, stream }
struct CompletionResponse { text, prompt_tokens, completion_tokens }
struct GpuInfo { vendor, vram_mb, supported }

impl InferenceEngine {
    fn load(model_path, n_gpu_layers) -> Result
    fn complete(&self, req: CompletionRequest) -> Result
    fn complete_stream(&self, req, tx: mpsc::Sender) -> Result<()>  // token 流
    fn detect_gpu() -> GpuInfo
    fn token_count(text: &str) -> usize  // GGUF tokenizer
}
依赖: llama-cpp-2, tokio, openloom-models
Feature flags: cuda, metal, vulkan (workspace 级透传)

2. crates/router/ — 智能路由

职责 & API
职责: 意图分类 + 复杂度评分 + 技能匹配

公开类型:
struct RouterConfig { model_path, keyword_rules, fallback_threshold }
struct ClassifyOutput {
    intent: Intent,          // 意图标签
    complexity: f32,         // 0.0 ~ 1.0
    skill_match: Option,  // 匹配到的 skill 名
    confidence: f32,         // 分类置信度
    target_model: TargetModel,
}
enum Intent { Chat, FileOperation, WebSearch, CodeAssist, Schedule, ... }
enum TargetModel { Local, Cloud, None }  // None = skill 直接处理

impl SmartRouter {
    fn new(config: RouterConfig, engine: Arc) -> Self
    fn classify(&self, text: &str) -> ClassifyOutput
    fn register_skill_triggers(&mut self, skill_name, triggers: Vec)
}
内部流程: 关键词匹配(零token) → 命中→高置信度直接返回 → 未命中→调 Qwen3-1.7B 分类 → 置信度 < 0.7 → 降级标记
依赖: openloom-inference, regex, openloom-models

3. crates/skills/ — 技能引擎

职责 & API
职责: Skill 注册/发现/执行 + CLI Bridge 工具发现

公开类型:
trait Skill: Send + Sync {
    fn name() -> &'static str
    fn manifest() -> SkillManifest      // triggers, permissions, context
    fn invoke(&self, params: Value) -> Result
    fn context_md() -> &'static str     // <= 200 tokens
}

struct SkillManifest { triggers: Vec, permissions: Permissions, ... }
struct SkillRegistry { ... }
struct CliBridge { ... }

impl SkillRegistry {
    fn register(&mut self, skill: Box)
    fn find_by_trigger(&self, text: &str) -> Option<&dyn Skill>
    fn list_all() -> Vec
    fn invoke(name, params) -> Result
}

impl CliBridge {
    fn discover_path_tools() -> Vec  // 扫描 PATH 可执行文件
    fn parse_help(binary: &str) -> CliTool    // --help 输出解析
}
5 个内置 Skill: FileManager, InfoRetriever, ScheduleReminder, CodeAssistant, WebBrowser
依赖: wasmtime (Phase 2), serde_json, which/where, openloom-models

4. crates/engine/ — 编排引擎 (轻量)

职责 & API
职责: EventBus + 请求派发 + 生命周期管理

公开类型:
struct Engine {
    router: SmartRouter,
    skills: SkillRegistry,
    inference: Arc,
    memory: MemoryPipeline,
    event_bus: broadcast::Sender,
}

enum EngineEvent { CognitionUpdated(...), TokenUsed(...), StateChanged(...), Error(...) }

impl Engine {
    fn new(config: EngineConfig) -> Result
    fn start() -> Result>  // 启动后台任务
    fn handle_message(&self, msg: ChatMessage, session_id) -> Result
    fn shutdown() -> Result<()>
}
请求流程: handle_message → router.classify → skill.invoke 或 inference.complete → 返回 → 后台 memory.process
依赖: openloom-router, openloom-skills, openloom-inference, openloom-memory, tokio

5. crates/server/ — HTTP + WSS 服务

职责 & API
职责: Axum HTTP + WebSocket + SSE + JSON-RPC 2.0

端点:
GET  /health              → { status, uptime, gpu_info }
GET  /ws                  → WebSocket upgrade (JSON-RPC 2.0 帧)
GET  /sse/{session_id}    → SSE token 流 (前端直连)
POST /api                 → JSON-RPC 2.0 (备选 HTTP fallback)
JSON-RPC 方法 (Phase 1 实现): chat.send, skill.invoke, skill.list, memory.query, memory.persona, agent.status, cache.stats, config.get, config.set, system.health, system.shutdown

服务端通知: cognition.updated, token.usage, error
依赖: axum, tower, tokio-tungstenite, serde_json, openloom-engine

server 依赖 engine（轻量编排），不直接依赖 router/skills/inference
memory 作为独立管线，被 engine 异步调用
